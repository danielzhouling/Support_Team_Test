# Dmall Helpdesk 技能评估系统 - 项目全程讨论记录

> 本文档记录项目所有需求讨论、技术决策、设计方案，以便未来随时查阅。

## 一、项目背景

### 1.1 产品定位
- **产品名称**：Dmall Helpdesk 技能评估系统
- **目标用户**：Dmall 菲律宾 SM 项目 Helpdesk 团队
- **核心价值**：AI辅助出题 + 在线考核 + 智能分析

### 1.2 用户规模
| 角色 | 人数 | 说明 |
|------|------|------|
| 超级管理员 | 1人 | 系统初始创建，负责系统配置 |
| 管理员 | 1-3人 | Daniel 等，负责日常运营 |
| 员工 | 11人 | Helpdesk 团队成员，参与考核 |

## 二、技术选型决策

### 2.1 技术栈
| 层级 | 技术选型 | 决策理由 |
|------|----------|----------|
| 前端 | Vue3 + Vite + Element Plus | 稳定、文档完善、组件丰富，适合后台系统 |
| 后端 | Python FastAPI | 轻量、异步性能好、与AI SDK集成方便 |
| 数据库 | SQLite → PostgreSQL | 初期零配置，后期无缝迁移 |
| AI能力 | Minimax 2.7 国际版 API | 用户提供key，稳定国际版服务 |

### 2.2 选型依据
- **稳定优先**：团队内部使用，不是大面积推广
- **快速开发**：技术栈成熟，文档完善
- **AI集成**：用户已有Minimax API key

## 六、PRD核心功能摘录

### 6.1 系统功能架构
- 账号管理模块（员工CRUD/批量导入/状态管理）
- 文档管理模块（上传/分类/预览/AI解析）
- 题库管理模块（题目CRUD/AI生成/审核/导入导出）
- 考核管理模块（创建/发布/监控/批阅/成绩发布）
- 数据报表模块（团队看板/个人档案/数据导出）
- 员工端模块（双入口/考核作答/学习中心）
- Chat智能体（对话答疑/快捷操作）
- AI能力（文档解析/题目生成/报告生成）

### 6.2 考核管理核心流程
```
创建考核（3步向导）
├── Step1: 基本信息（名称/及格分/时长/截止时间）
├── Step2: 题目配置（从题库选择/AI智能选题）
└── Step3: 参与人员（指定团队/指定人员/导入名单）
    ↓
发布考核 → 员工作答 → 交卷 → 批阅（问答题）→ 成绩发布
```

### 6.3 员工端考核流程
```
考核列表 → 考核说明 → 开始作答 → 交卷确认 → 成绩展示
              ↓
        作答页（单选/多选/判断/问答）
        ├── 顶部：计时器 + 进度
        ├── 题目区域：题目内容 + 选项
        └── 底部：题号导航 + 交卷按钮
```
